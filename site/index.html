THE FACTORY'S COMPOSITION LAYER

One fleet, built as separate products. Weave is what makes them cohere.

Misty Step ships a fleet of independently-built agent products — a work ledger, a production-health service, an eval workbench, an execution plane, a live agent viewer, and more. Weave is the layer underneath all of them: the versioned event contracts, the cross-repo doctrine (stacked-diff review, the five-faces interface law), and — because someone has to say what the fleet actually is — this page, the map of every product in it.

Every claim on this page is backed by a pinned schema, a merged doctrine doc, or the fleet coverage matrix — linked below, not asserted.

What Weave actually is

contracts Versioned event schemas. proof Pass/fail fixtures and a CI conformance gate.
doctrine Rules that outlive one repo. shape A schema/doctrine repo, not a runtime UI.
trust Public repo, public doctrine. fit Aesthetic tokens with the Weave mark.

The two contracts today are weave.remote_event.v1 and weave.work_item_proposal.v1; the doctrine docs are the five-faces interface law and the stacked-diff review discipline.

The family

Every product Weave composes. Capability signals are pulled straight from the fleet coverage matrix where one exists; newer products get a plain description instead of a guessed grade. Cards link out once a product has a live public site — most don't yet, and this page says so.

Bitterblossom

The Weave execution plane: backlog chewing, review dispatch, incident triage, and ad-hoc repo work, dispatched and observed as agent runs.

API + CLI + UI shipped, MCP read-only by construction — no live public site yet.

Roster

Makes agents first-class citizens of the factory: the one place every agent's role, permitted skills, and model policy is declared, then run on any plane.

New repo, predates the fleet coverage pass — no face-coverage claim made here, no live site yet.

Powder

The self-hosted work ledger: cards, claims, runs, comments, and audit history that operators and agent teams share instead of losing state in chat.

Live site → API + CLI + MCP shipped; UI (Kanban board) is tracked backlog.

Glass

The live stage: agents publish typed surfaces — HTML, diffs, traces, terminal output — while they work, and the operator replies without breaking their flow.

Public site is built and ready; the repo is private today, so no live link — publishing needs a visibility decision this page doesn't make.

Glance

Turns a repository into a walkable, cited HTML understanding site — one narrated page per directory, every claim traceable to source.

Private repo, greenfield Rust rewrite (successor to the archived Go tool) — no face-coverage claim, no live site yet.

Canary

The agent-first command center for production health: error logging, uptime checks, incidents, and operational events that agent fleets read from and write back to.

API + CLI + MCP shipped; TypeScript SDK is built and tested, pending npm publish. No live site yet.

Crucible

The eval and benchmark workbench: turns "we need an eval for this" into an auditable task definition, grader mix, and result surface.

CLI + MCP shipped; API is localhost-only by design. Private repo, no live site yet.

Exocortex

A public, local-first evidence core: an agent asks for context instead of rummaging a private world, and a human can audit exactly what it saw.

New repo, predates the fleet coverage pass — no face-coverage claim made here, no live site yet.

Landmark

Public open-source release intelligence: changelog synthesis, version bumps, and release evidence from a Rust CLI that also runs as a GitHub Action.

CLI + GitHub Action + crates.io SDK shipped; API is a ratified no-server exception. No live site yet.
